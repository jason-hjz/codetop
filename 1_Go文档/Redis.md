Remote Dictionary Service 键值对数据库服务器

- 定义：一个内存数据库，用作SQL的缓存



架构

- 主从复制：
  - 部分同步
  - 全量同步：主实例先发送RDB快照给副本，再发送两个偏移量之间的中间更新指令

- 哨兵：
  - 监控：确保主从实例按预期工作
  - 通知：通知系统管理员Redis实例中的事件
  - 故障转移管理：如果主实例不可用且足够多的节点（quorum）同意，哨兵节点可以启动故障转移
  - 配置管理：哨兵节点可告诉客户端当前的主实例是什么

- 集群：
  - 将存储的数据分散到多台机器，称为分片
  - 使用Hashslot哈希槽，一共16384个，所有的数据根据Key的哈希值对应到槽 CRC16(key) % 16384；当加入新的分片，调整让每个分片平均持有槽数。
- Gossiping协议
  - 节点不断通过通信了解哪些分片可用并准备好，当足够多分片同意M1不响应，可以决定将M1的副本S1提升为主节点



持久化

- RDB Redis数据库 快照

- AOF Append Only File：记录服务器接收到的每个写入操作，这些操作将在服务器启动时再次被执行

- Fork：当fork一个进程时，父进程和子进程共享内存，并且在该子进程中Redis开始快照进程，这是通过一种称为写时复制的内存共享技术实现的

  > Redis 快照时 fork 子进程，利用 COW 让父子进程共享内存：子进程只读生成快照，父进程写操作触发内存页复制，不修改的部分不复制，仍然父子共享；既不阻塞主进程，又保证快照数据的一致性；
