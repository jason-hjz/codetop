### db/mysql/conn.go

这段代码是一个 Go 语言编写的 MySQL 数据库连接工具包（包名 `mysql`），核心作用是：

1. 初始化并维护一个全局的 MySQL 数据库连接池
2. 提供获取数据库连接的方法
3. 提供通用的查询结果解析方法，将 SQL 查询结果转换为易读的 `[]map[string]interface{}` 格式
4. 包含简单的错误检查工具函数

其他业务代码可通过 `mysql.DBConn()` 获取连接池，执行 `Query`/`Exec` 操作后，用 `mysql.ParseRows` 解析结果。



### db/file.go

这段代码封装了云存储系统中「文件元信息」的数据库操作逻辑，核心能力包括：

1. 文件上传完成后，将文件的元信息（哈希、名称、大小、存储地址）保存到 MySQL 的 `tbl_file` 表中；
2. 根据文件哈希查询单个文件的元信息；
3. 批量查询指定数量的文件元信息；
4. 定义了与数据库 `tbl_file` 表对应的结构体，用于映射查询结果。



### db/user.go

这段代码属于 `db` 包，是分布式云存储系统「用户模块」的数据库交互层，基于之前封装的 MySQL 连接池（`mydb`）实现了用户全生命周期的核心数据库操作：

1. 用户注册（避免重复用户名）；
2. 登录密码验证（对比数据库中的加密密码）；
3. 刷新用户登录 Token（存在则更新，不存在则插入）；
4. 查询用户基本信息（用户名、注册时间）；
5. 高效检查用户是否存在。

### handler/auth.go

这段代码是 Go 语言 Gin Web 框架的**权限验证中间件（拦截器）**，归属 `handler` 包。它会在请求到达业务处理器前执行，校验请求中的「用户名长度」和「访问 Token 有效性」：

- 验证通过：继续执行后续的业务逻辑；
- 验证失败：终止请求流程，返回 401（未授权）JSON 响应。



### handler/mpupload.go

分布式云存储系统中**大文件分块上传**的核心处理器（Handler），基于 Gin 框架实现了分块上传的完整流程：初始化上传、上传单个分块、完成上传（校验分块 + 合并 + 更新数据库），同时结合 Redis 缓存分块上传状态，保证分块上传的完整性。这段代码的核心流程：

1. **初始化上传**：接收文件基本信息（哈希、大小），生成唯一 UploadID，计算分块大小 / 数量，将状态存入 Redis；
2. **上传单个分块**：接收指定 UploadID 的分块数据，保存到本地磁盘，更新 Redis 中该分块的上传状态；
3. **完成上传**：校验 Redis 中所有分块是否上传完成，TODO 合并所有分块，最后将文件元信息写入 MySQL 数据库。



### handler/upload.go

这段 Go 代码是一个基于 Gin 框架的**分布式云存储系统的 HTTP 处理器（Handler）**，主要实现了文件上传（含秒传）、文件元信息查询、文件下载、文件重命名、文件删除等核心功能，并且支持将文件存储到本地、Ceph、阿里云 OSS 三种存储介质，同时会将文件元信息和用户文件关联关系持久化到数据库中。



### handler/user.go

这段 Go 代码是分布式云存储系统的**用户认证处理器（Handler）**，核心实现了用户注册、登录、用户信息查询、用户名唯一性校验等功能，同时包含密码加密、Token 生成 / 验证等安全相关逻辑，是整个系统的用户身份管理核心。

这段代码是典型的 Web 系统用户认证模块，核心关键点如下：

1. **密码安全**：采用「盐值 + SHA1」哈希加密，避免明文存储，符合安全最佳实践；
2. **Token 鉴权**：登录生成唯一 Token，替代用户名密码做后续接口鉴权，但 Token 验证逻辑未完全实现；
3. **交互设计**：前后端通过 JSON 交互，包含状态码、提示信息、跳转路径，适配前端页面逻辑；
4. **待完善点**：Token 时效性验证、用户信息接口的 Token 鉴权、异常处理（如数据库错误）需补充。



### meta/filemeta.go

这段 Go 代码是分布式云存储系统的 `meta` 包（文件元信息模块），核心实现了**文件元信息的内存管理**和**数据库持久化管理**，定义了文件元信息的结构体，并提供了新增 / 更新、查询、批量查询、删除等操作接口，同时兼顾了内存（临时存储）和 MySQL（持久化存储）两种存储方式。

1. **核心结构体**：`FileMeta` 定义了文件元信息的核心字段，是整个模块的数据载体；
2. **内存存储**：用 `map[string]FileMeta` 存储文件元信息（Key 为文件 SHA1 哈希），适合高频读写、临时存储；
3. **数据库存储**：通过调用 `mydb` 包（数据库层）接口，将元信息持久化到 MySQL，保证数据不丢失；
4. **设计思路**：内存 + 数据库双存储，内存提升查询性能，数据库保证数据持久化。
